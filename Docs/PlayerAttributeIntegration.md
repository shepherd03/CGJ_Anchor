# 角色属性接入进度表

这份文档用于判断 `Config/Luban/Datas/#game.playerAttribute.xlsx` 中声明的属性是否已经进入实际游戏链路。

核心判断：

- 已声明：属性表有行，且 `CharacterAttributeIds` 有常量。
- 已初始化：新游戏时会写入 `CharacterAttributeSet`。
- 已读取：流程、公式、UI、事件、Buff 或商店至少有一处读取该属性。
- 已影响结果：该属性会改变行动、资源、周结算、月结算、事件触发、Buff 效果或结局。
- 已验证：能通过自动流程、手动流程或测试面板看到可复现结果。

## 状态定义

```text
未接入    表和常量存在，但运行时没有读取，改表值不会影响游戏。
待验证    已有代码读取或写入，但影响范围不完整，或者缺少可复现验收。
已接入    属性会稳定影响游戏链路，并有明确验收方式。
记录型    属性会被写入或展示，但目前不反向影响其他核心结果。
暂缓      设计上可能需要，但依赖事件、Buff、商店或 UI 等后续系统。
```

## 当前可优先接入项

### 第一批：最小风险，直接接公式

这些属性不需要新增 UI 或新表，只要在现有 `GameFlowResolveService` 和 `GameFlowBlackboard.ApplyWeekResult` 附近接入即可。

```text
已完成：

1028 每周Bug值变化
1029 每周画面值变化
1030 每周氛围值变化
1024 愿望单增长百分比加成
```

建议原因：

- 1028/1029/1030 在周结算中作为固定修正加入。
- 1024 在月结算中作为愿望单增长百分比加入。
- 百分比倍率最低钳制为 0，避免负增长倍率把月结算愿望单反向放大。

### 第二批：利用现有周行动记录

已完成。它们基于 `ActionAllocations`、全局周序号和结算后的属性值判断，不需要新增 UI。

```text
1022 3/6/9回合结束愿望单奖励
1023 低Bug每回合愿望单奖励
1025 高画面回合结束愿望单增长加成
1026 三房间同回合行动愿望单增长加成
```

建议原因：

- 1022 使用 `TotalWeekIndex`，不会被每月 `WeekIndex` 重置影响。
- 1023 使用周结算后的 Bug 值，阈值为低于 40。
- 1025 使用周结算后的画面值，阈值为高于 60。
- 1026 对应当前三房间：`Program / Art / Audio`。

### 第三批：需要补运行时历史或选择入口

这些需要先明确“房间”和当前 `GameDevelopmentTrack` 的对应关系，或记录上周行动。

```text
1011-1019 房间行动收益
1020 连续同房间行动愿望单奖励
```

注意：

- 当前代码的行动方向是 `Program / Art / Audio`，对应程序、美术、音效三房间。
- 如果接 `1020`，需要在黑板里记录上一周主投房间或上一周有投入的房间集合。

### 第四批：依赖商店/事件/Buff UI

这些适合等选择界面或效果应用器明确后再接。

```text
1021 回合开始50%愿望单倍率
1027 每回合固定强化费用
gameEvent 表效果
buff 表效果
```

注意：

- `1027` 已经有 `BudgetShopState` 状态入口，但目前没有购买消息、费用校验和属性应用。
- `1021` 涉及随机触发，最好先确定是否需要可复现随机种子和 UI 提示。
- `gameEvent` 和 `buff` 表现在只有数据和校验，没有运行时应用器。
